This is the Beam Bots project. Its core library is located in the `bb` subfolder, and all the ecosystem support packages are in the `bb_*` subfolders. The website is in the `website` subfolder. All these projects are on GitHub in the `beam-bots` organisation namespace.

## Blog Post

Please analyse the website to find the date of the last published progress update (you can find blog posts with the tag `progress` in their front matter). Then analyse the changelogs and the contents of any merged pull requests to all the beam bots ecosystem projects since that date. Additionally, ask me if there are any important events or updates I would like to highlight (hardware progress, personal circumstances, community news, etc.).

Then draft a new blog post for the website outlining the progress since the last progress update. Include today's date and the `progress` tag in the front matter along with a jaunty-but-relevant title and description. Put the interesting or noteworthy information at the top of the post.

Examples of progress updates to include:
- New package releases (check hex.pm via GitHub releases)
- Upstream dependency releases that unblock our work
- Breaking changes that users need to know about
- Project updates from framework users
- Videos and articles about the framework or relevant events
- Hardware milestones or demos

My writing style is light, direct and no nonsense with a few dad jokes and some actual humour thrown in for good measure.

Treat the upstream GitHub repos as sources of truth as the local repositories may contain work in progress or unreleased packages. Don't include website updates in the post unless I specifically tell you.
